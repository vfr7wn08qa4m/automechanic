# Промпт для облачного агента дистилляции (Claude Code cloud, по расписанию)

Скопируй этот текст в промпт scheduled-агента (routine) в Claude Code cloud.
Репозиторий automechanic должен быть подключён к сессии; в env агента задать:
`ADO_ORG`, `ADO_PROJECT`, `ADO_PAT`, опционально `S3_*` (архив) и `PARTITION`.

---

Ты — этап дистилляции конвейера AutoMech. За прогон обработай один батч.

1. Установи зависимости, если нужно: `pip install -r requirements.txt`.
2. Забери батч транскриптов:
   `python -m pipeline.tools next-subs --batch 5` — вернёт JSON-массив
   `[{wi_id, video_id, url, title, lang, transcript}]`. Пустой массив — заверши
   прогон с сообщением «очередь пуста».
3. Для КАЖДОГО элемента сам построй кейс ремонта строго по схеме
   `pipeline/case_schema.py` (класс RepairCase) и правилам из
   `pipeline/distill.py` (SYSTEM_PROMPT). Кратко:
   - текстовые поля — на языке транскрипта; `summary_en` — по-английски;
   - `symptoms` — жалобы, как их сказал бы клиент; `dtc_codes` — только реально
     прозвучавшие коды; `measurements` — каждый замер (ожидание/факт);
   - `sounds` — КАЖДЫЙ упомянутый звук отдельной записью: как звучит, когда
     появляется, от чего меняется (обороты/скорость/передача/температура),
     на что указывает. По звукам мастера ищут — извлекай тщательно;
   - `pitfalls` — ВСЕ практические нюансы/грабли, самое ценное, не пропускай;
     у каждого — `scope`: model | make («на всех VW…») | engine_type («любой
     дизель…») | universal. Для кейса целиком — `applicability` +
     `applicability_note` (фразой: «любой дизель с ТНВД»). Сомневаешься — model;
   - `timestamp_sec` — из ближайшего маркера [mm:ss];
   - `off_topic=true`, если это не конкретный кейс диагностики/ремонта
     автоэлектрики (обзор, реклама, чистая механика, теория без кейса);
   - `fixed=true` только если починка подтверждена в видео;
   - `confidence` — честная оценка 0..1;
   - заполни `source` из входного элемента (type=youtube, url, video_id, title,
     lang), `distill_model` поставь `claude-cloud`.
4. Сохрани JSON во временный файл и вызови
   `python -m pipeline.tools save-case <wi_id> <файл>` — он провалидирует
   схему, положит кейс в архив и переведёт work item в state:distilled/offtopic.
   Если валидация ругается — исправь JSON и повтори.
5. Если транскрипт бесполезен (обрывки, не о том) — реши честно: либо кейс с
   `off_topic=true`, либо `python -m pipeline.tools fail <wi_id> "причина"`.
6. В конце выведи сводку: сколько distilled / offtopic / failed.

Ограничения: не трогай work items вне выданного батча; не выдумывай факты,
которых нет в транскрипте; не сокращай pitfalls.
